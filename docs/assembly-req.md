# eSIM 集成商平台

## 一、核心功能总结
本系统为 **eSIM 集成商平台**，用于集成商进行eSIM业务全流程管理与服务支撑，实现从数据概览、eSIM 套餐选购、资产管控到订单财务、团队协作、技术对接的全流程闭环管理，是面向集成商的 eSIM 业务运营与技术集成中枢。

---

## 二、业务流程
### （一）成为集成商
1. 组织或个人通过线上渠道、销售对接等方式达成合作意向，向产品运营提交信息申请成为平台集成商
2. 产品运营根据申请完成集成商的信息配置，如配置价格、供应商、支付方式等
3. 组织或个人从销售或产品运营获取平台登录凭证，成为eSIM集成商

### （二）平台对接
1. 集成商登录平台，在线支付或与产品运营交涉完成充值
2. 进入开发者配置页，获取、更新 API 配置及回调配置
3. 参考开发文档，结合自身业务，通过API查询与订阅套餐

### （三）账单结算
 - 预付模式：集成商户可在线或通过销售/产品运营线下完成充值，充值金额转为账户可用余额
 - 后付模式：产品运营按约定周期结算已使用金额并生成账单

## 三、详细需求描述
### （一）仪表盘（Dashboard）
1.  **数据总览展示**
    -   实时呈现当日核心业务数据，拆分**总订单**（金额、数量、取消数量）与**已激活订单**（金额、数量），并细化**基础订单**和**充值订单**的金额、数量明细。
    -   支持按时间维度（如“Today”）切换，查看不同时段的业务概览。

2.  **采购趋势分析**
    -   支持按「近7天/15天/30天」切换时间维度，以图表形式可视化展示 eSIM 采购量变化趋势。

3.  **活跃 eSIM 概览（Active eSIM Overview）**
    -   支持按「Last 7 days / Last 15 days / Last 30 days」切换时间维度，以折线图形式展示每日 eSIM 激活数量趋势，横轴为日期，纵轴为激活数量，清晰呈现激活量变化。
    -   图表默认展示近7天数据，支持切换至15天或30天维度，便于运营人员进行多周期趋势对比。

4.  **新购 eSIM 排行（New eSIM Purchased Top 10）**
    -   支持按时间维度（如 Today / Yesterday / This Week 等）筛选，展示新购 eSIM 数量排名前十的套餐或地区。
    -   无数据时显示「No Data.」，有数据时以列表或柱状图形式呈现排名、名称及对应数量，辅助识别热门套餐与高需求区域。

5.  **全局交互与信息展示**
    -   在页面顶部固定展示当前登录用户信息（姓名、邮箱）及账户实时余额，实现全局可见。
    -   提供页面快速返回顶部按钮，提升长页面浏览体验。

---

### （二）eSIM 套餐（eSIM Plans）
1.  **套餐多维度筛选**：支持通过**地区、套餐名称、有效期（天数）、数据类型、流量规格（GB）** 组合筛选套餐。
2.  **套餐详情呈现**：以列表形式展示套餐完整信息，包含名称、售价、流量额度、有效期、单位流量单价、建议零售价、网络速度、适用地区，同时配置选购、收藏操作入口。
3.  **结算与导出**：实时计算账户剩余额度、已选套餐数量、总计金额，提供「结算（Check out）」「价格查询（Pricing）」按钮，支持套餐数据导出为 CSV/Excel 格式。

---

### （三）我的 eSIM（My eSIM）
1.  **eSIM 精准查询**：支持通过**eSIM 交易号、套餐名称、eSIM 状态、SMDP 状态、订单号、商户交易号、ICCID** 多条件组合查询 eSIM 资产。
2.  **资产信息管理**：以列表形式展示 eSIM 全生命周期信息，包括套餐名称、eSIM 状态、绑定设备、剩余流量、剩余时长、激活截止时间，配套对应操作入口。
3.  **资产数量统计**：在列表顶部明确统计 eSIM **总数量**与**可用数量**。

---

### （四）我的订单（My Order）
1.  **订单筛选功能**：支持通过**订单号（批次号）、采购日期（起始日期-结束日期）** 筛选订单。
2.  **订单列表展示**：以列表形式呈现订单核心信息，包含订单号（批次号）、eSIM 数量、创建时间、订单金额、订单状态。
3.  **分页管理**：默认按10条/页展示订单，支持页码切换、指定页码跳转功能。

---

### （五）财务（Billing）
1.  **余额管理**：可视化展示账户余额，提供「充值（Top up）」按钮，配置**自动充值**功能开关及“编辑设置”入口。
2.  **交易记录筛选**：支持通过**交易 ID、交易类型、业务类型、时间区间（起始日期-结束日期）** 筛选交易记录。
3.  **财务数据展示**：以列表形式呈现交易详情，包含交易 ID、交易金额、交易类型、账户余额、交易时间、业务类型、备注。
4.  **财务导出与溯源**：支持交易记录、对账单（Statements）导出，提供「支付历史（Payment history）」查看入口，默认按10条/页分页展示。

---

### （六）团队成员（Team Members）
1.  **成员筛选与重置**：支持通过**邮箱、角色、手机号**精准搜索团队成员。
2.  **成员核心操作**：支持管理员执行「新增成员（Add Member）」「编辑成员信息」「删除成员（Remove）」「变更管理员（Change Admin）」操作。
3.  **角色与额度管理**：
    -   预设5种核心角色：**Admin（管理员）、Operator（操作员）、Customer Support（客服）、Finance（财务）、Developer（开发者）**，支持成员角色指定；
    -   明确团队成员上限为20人，实时统计当前成员数量（如「Members: 2 / 20」）；
    -   提供角色权限说明表，清晰展示各角色可访问的页面范围。
4.  **成员信息展示**：以列表形式呈现成员姓名、邮箱、手机号、角色及操作项，清晰展示团队权限架构。

---

### （七）开发者（Developer）
1.  **API 密钥管理**：系统自动生成 API 密钥（AccessCode、SecretKey），展示密钥名称、令牌（脱敏显示）、创建时间，提供「复制」功能；支持「重置密钥（Roll Keys）」，保障接口调用安全。
2.  **开发文档指引**：提供 API 文档专属访问链接，便于开发者查阅接口规范、调试对接。

3.  **Webhooks 配置**：支持开发者填写服务器事件通知 URL，可选配置白名单，提供「编辑（Edit）」功能，实现系统事件与第三方服务的实时联动。

---

### （八）用户信息（User Profile）
1.  **基础信息管理**：支持管理员查看并编辑个人核心信息，包括姓名、邮箱、手机号码，提供「编辑（Edit）」功能。
2.  **账户安全设置**：独立提供「修改密码（Change Password）」入口，保障账户登录安全。
